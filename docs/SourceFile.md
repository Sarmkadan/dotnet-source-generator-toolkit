# SourceFile

Represents a parsed C# source file with metadata and analysis capabilities. Used primarily in source generators and tooling to inspect and process C# files, track their state, and extract structural elements such as namespaces, types, and usings.

## API

### `Id`
A unique identifier for the source file. Typically derived from the file path or assigned by the toolkit. Used to track and reference the file across operations.

### `FilePath`
The absolute or relative file system path to the source file. Used to locate and reload the file when needed.

### `FileName`
The name of the file, including extension (e.g., `Program.cs`). Derived from `FilePath` but provided for convenience.

### `FileContent`
The full textual content of the source file as a string. May be updated during analysis or generation workflows.

### `ProjectPath`
The file system path to the project containing this source file. Useful for resolving project-level references or configuration.

### `Usings`
A list of `using` directives found in the file (e.g., `System`, `System.Collections.Generic`). Populated during parsing and modifiable via `AddUsing`.

### `Namespaces`
A list of top-level or nested namespace declarations in the file. Includes both root and nested namespaces (e.g., `MyApp`, `MyApp.Services`).

### `TypeNames`
A list of type names (class, struct, interface, enum, record) declared in the file. Includes fully qualified names where applicable.

### `Attributes`
A list of attribute names (e.g., `Obsolete`, `Serializable`, `JsonSerializable`) applied to types or members in the file.

### `LineCount`
The total number of lines in the file, including empty and comment lines. Updated during parsing or content changes.

### `FileSizeBytes`
The size of the file in bytes. Updated when content is modified.

### `FileType`
An enum value indicating the type of source file (e.g., `Code`, `Config`, `Generated`). Used to categorize files during processing.

### `Status`
An enum value representing the current state of the file (e.g., `Parsed`, `Analyzed`, `Modified`, `Error`). Tracks lifecycle and validity.

### `CreatedAt`
The timestamp when the `SourceFile` instance was created or the file was first parsed.

### `ModifiedAt`
The timestamp of the last modification to the file content or metadata. Updated automatically on content changes.

### `GeneratedFrom`
An optional identifier (e.g., generator name or template ID) indicating the source of generated content. `null` for non-generated files.

### `IsGenerated`
A boolean indicating whether the file content was generated by a source generator or tool. Used to skip or validate generated files.

### `ParsedMetadata`
A dictionary of additional metadata extracted during parsing (e.g., compiler version, target framework, conditional symbols). Values are of type `object` and may include strings, booleans, or structured data.

### `AnalyzeContent()`
Parses the file content and populates metadata such as `Usings`, `Namespaces`, `TypeNames`, `Attributes`, `LineCount`, and `FileSizeBytes`. Updates `Status` to `Analyzed` on success; sets `Status` to `Error` and may throw on parse failure.

> **Throws**: `InvalidOperationException` if `FileContent` is `null` or empty.
> **Throws**: `FormatException` if parsing fails due to malformed C# syntax.

### `AddUsing(string usingDirective)`
Adds a `using` directive to the `Usings` list and updates the internal state. The directive is not written back to disk unless explicitly saved by the consumer.

> **Parameters**:
> - `usingDirective` (string): The full `using` statement (e.g., `using System.Text;`).
>
> **Throws**: `ArgumentNullException` if `usingDirective` is `null`.
> **Throws**: `ArgumentException` if `usingDirective` is empty or not a valid `using` statement.
