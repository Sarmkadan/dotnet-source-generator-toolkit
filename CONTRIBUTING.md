# Contributing to dotnet-source-generator-toolkit

We welcome contributions! Please follow these guidelines:

1. **Fork** the repository and create a feature branch (`git checkout -b feature/name`).
2. **Build and Test**: Ensure everything passes locally (`dotnet build && dotnet test`).
3. **Standards**:
   - Follow C# coding conventions and existing `.editorconfig`.
   - Provide XML documentation for all public APIs.
   - Maintain author headers at the top of files.
4. **Pull Requests**:
   - Keep PRs focused on one feature/fix.
   - Reference related issues.
   - Ensure all tests pass.
5. **Issues**: Use GitHub Issues for bugs/features with clear reproduction steps.
