# GenerationOptions Usage Examples

The `GenerationOptions` record provides a centralized way to control cross-cutting emission policies across all code generators in the dotnet-source-generator-toolkit.

## Overview

`GenerationOptions` controls:
- Nullable context directives (`#nullable enable`/`#nullable disable`/`#nullable restore`)
- Namespace declaration style (braced vs file-scoped)
- `[GeneratedCode]` attribute emission
- Custom file headers
- Target C# language version
- Indentation size

## Basic Usage

### Default Options

```csharp
var options = GenerationOptions.Default;
// Uses: #nullable enable, braced namespaces, [GeneratedCode] attribute, 4-space indent
```

### Minimal Options

```csharp
var options = GenerationOptions.Minimal;
// Uses: No nullable context, no headers, no [GeneratedCode] attribute
```

### File-Scoped Namespaces

```csharp
var options = GenerationOptions.FileScoped;
// Uses: File-scoped namespaces (namespace MyNamespace;)
```

## Advanced Configuration


### Custom Options

```csharp
var options = new GenerationOptions(
    NullableContext = NullableContext.Disable,
    NamespaceStyle = NamespaceStyle.FileScoped,
    EmitGeneratedCodeAttribute = GeneratedCodeAttributeMode.Conditional,
    HeaderComment = "Custom Header Comment",
    LangVersion = "12.0",
    IndentSize = 2
);
```

### Using with CodeEmitter

```csharp
using DotNetSourceGeneratorToolkit.Domain;
using DotNetSourceGeneratorToolkit.Services;

// Generate a complete source file with options
var options = new GenerationOptions(
    NullableContext = NullableContext.Enable,
    NamespaceStyle = NamespaceStyle.Braced
);

var sourceCode = CodeEmitter.GenerateSourceFile(
    options,
    @"MyNamespace",
    "public class MyClass { }",
    new[] { "System.Collections.Generic" },
    "My Company",
    "Example class"
);

Console.WriteLine(sourceCode);
```

### Using with TemplateEngineService

```csharp
var templateEngine = new TemplateEngineService(fileSystemService, logger);
var context = new Dictionary<string, object> {
    { "ClassName", "User" },
    { "Properties", new[] { "Id", "Name", "Email" } }
};

var options = new GenerationOptions(
    NullableContext = NullableContext.Enable,
    HeaderComment = "User Entity Template"
);

var rendered = await templateEngine.RenderAsync(
    templateContent,
    context,
    options
);
```

## MSBuild Configuration


You can configure GenerationOptions via MSBuild properties in your `.editorconfig` or project file:

### .editorconfig

```ini
[*.cs]
# Nullable context for generated files
dotnet_diagnostic.SGTK001.preferred_nullable_context = enable

# Namespace style
dotnet_diagnostic.SGTK002.preferred_namespace_style = file_scoped

# GeneratedCode attribute mode
dotnet_diagnostic.SGTK003.preferred_generated_code_attribute = conditional

# Custom header comment
dotnet_diagnostic.SGTK004.preferred_header_comment = Generated by MyToolkit

# Indentation size
dotnet_diagnostic.SGTK005.preferred_indent_size = 2
```

### Project File

```xml
<PropertyGroup>
    <LangVersion>12.0</LangVersion>
</PropertyGroup>

<ItemGroup>
    <AnalyzerConfig Include=".editorconfig" Link=".editorconfig" />
</ItemGroup>
```

## GenerationOptionsProvider


The `GenerationOptionsProvider` service reads MSBuild configuration and provides GenerationOptions:

```csharp
// In your DI configuration
services.AddSingleton<IGenerationOptionsProvider>(provider => 
    new GenerationOptionsProvider(
        provider.GetRequiredService<AnalyzerConfigOptionsProvider>(),
        provider.GetRequiredService<ILogger<GenerationOptionsProvider>>()
    )
);

// Usage in services
public class MyGeneratorService
{
    private readonly IGenerationOptionsProvider _optionsProvider;
    
    public async Task GenerateAsync()
    {
        var options = _optionsProvider.GetGenerationOptions();
        // Use options for generation
    }
}
```

## Examples

### Example 1: Disable Nullable Context

```csharp
var options = new GenerationOptions(
    NullableContext = NullableContext.None
);

// Generated file will NOT have #nullable enable/disable directives
```

### Example 2: File-Scoped Namespaces

```csharp
var options = new GenerationOptions(
    NamespaceStyle = NamespaceStyle.FileScoped
);

// Generated file will use: namespace MyNamespace;
// Instead of: namespace MyNamespace { ... }
```

### Example 3: Conditional GeneratedCode Attribute

```csharp
var options = new GenerationOptions(
    EmitGeneratedCodeAttribute = GeneratedCodeAttributeMode.Conditional
);

// Generated file will have:
// #if NET6_0_OR_GREATER
// [GeneratedCode("Toolkit", "1.0.0")]
// #endif
```

### Example 4: Custom Header

```csharp
var options = new GenerationOptions(
    HeaderComment = "Company Confidential - Internal Use Only"
);

// Generated file will have custom header instead of default
```

## Best Practices

1. **Use Default for most cases** - Provides sensible defaults that work well for most projects
2. **Use FileScoped for modern projects** - Reduces nesting and improves readability
3. **Keep nullable context enabled** - Helps catch null reference issues during development
4. **Use Conditional GeneratedCode** - Avoids unnecessary attributes in release builds
5. **Configure via .editorconfig** - Keeps configuration close to the code it affects

## API Reference

### GenerationOptions Properties

| Property | Type | Default | Description |
|----------|------|---------|-------------|
| `NullableContext` | `NullableContext` | `Enable` | Controls nullable directives |
| `NamespaceStyle` | `NamespaceStyle` | `Braced` | Controls namespace style |
| `EmitGeneratedCodeAttribute` | `GeneratedCodeAttributeMode` | `Always` | Controls [GeneratedCode] attribute |
| `HeaderComment` | `string?` | `null` | Custom header comment |
| `LangVersion` | `string?` | `null` | Target C# language version |
| `IndentSize` | `int` | `4` | Indentation size in spaces |

### NullableContext Enum

- `None` - No nullable directive
- `Enable` - Adds `#nullable enable`
- `Disable` - Adds `#nullable disable`

### NamespaceStyle Enum

- `Braced` - Traditional `namespace X { ... }`
- `FileScoped` - File-scoped `namespace X;`

### GeneratedCodeAttributeMode Enum

- `Never` - Never emit attribute
- `Always` - Always emit attribute
- `Conditional` - Conditionally emit with `#if` directive

## Integration with Existing Code

The GenerationOptions system is designed to be non-breaking. All existing code continues to work with default options. To adopt the new system:


1. Add `GenerationOptions` parameter to your generation methods
2. Use `CodeEmitter.GenerateSourceFile()` for complete file generation
3. Update templates to use the new rendering methods with options
4. Configure via MSBuild properties as needed

## Migration Guide

### Before (Hardcoded)

```csharp
var code = $@"#nullable enable
// ============================================
// Author: ...
// Generated Mapper
// ============================================

namespace MyApp.Mappers {{
    public class UserMapper {{
        ...
    }}
}}";
```

### After (Configurable)

```csharp
var options = new GenerationOptions(
    NullableContext = NullableContext.Enable,
    NamespaceStyle = NamespaceStyle.Braced
);

var code = CodeEmitter.GenerateSourceFile(
    options,
    "MyApp.Mappers",
    "public class UserMapper { ... }"
);
```
